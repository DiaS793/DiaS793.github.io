Mystery Educator


This project is called Mystery Educator. For this project, the ladies of my cohort were given the chance to be on the first women-only team. It was a lot of fun being able to have this opportunity to showcase what women can do when put in a team together.
The prompt of this project piggybacks off of a previous project the cohort did that was called JAC's Island Hopping website. For the project, we had to create a sing-page application (SPA) for kids that educated them on many different subjects, but with a twist, each time a child clicked on a subject they had to learn something different, hints the name Mystery Educator.
This project required that we rely heavily on 4 to 5 APIs picked by the instructors. We also had to create our own API to use along with them. We used Java and JavaScript to build the application as well as CSS and HTML. We also utilized JSON for data formatting.
My role in this was to create the contact form and work on the art section. The contact page was made using a simple grid box design. It does not do anything or go anywhere else on the website, but I do plan on going back into work on making it post to the back end.

For the art section, I utilized The Metropolitan Art Museum in New York City to create the page. Within each mystery box, I decided to break each one down into a few art categories, masks, music, architecture, and statues. When a child clicks on one of the boxes, the page will randomly generate a different piece from that mystery box.



The Mystery part of this project was a hard concept to understand and achieve. The museum's API was made to give JSON data on each individual piece of art. They were not grouped in a collection as I had hoped. So I wasn't sure how to tackle the Mystery part of the project with this API. However, after talking and working with my group and the instructors I was able to achieve this part of the project concept by creating an array with 7 of the API IDs for each piece I wanted to use.



There are 4 arrays broken into the names of the four pre-mentioned categories, Mask, Music, Architecture, and Statuses. Then I created an if statement with an event.target that called in an array, let's say artMask, and then set Math.floor and Math.random * the length of that array. Then set the value to artMask[artRand]. After this, I added the value to the API's URL in the apiHelpers.getRequest. I repeated this for each art ID. Then when a user clicks on a box, the IDs will randomly generate information from a different art ID each time.
I hope to go back to this project and add a bit more style to the pages to make them look my stylized.
